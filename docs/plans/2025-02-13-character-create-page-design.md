# 角色创建页面设计

## 概述

重新设计角色创建页面，将属性、职业、技能、背景、装备整合到单页长滚动布局，风格与 Dashboard 保持一致：干净、无背景色、黑白配色。

## 设计决策

| 决策点 | 选择 |
|--------|------|
| 页面布局 | 单页长滚动 |
| 属性投骰 | 点击直接显示数值，无动画，支持单独投骰 |
| 技能分配 | 点数分配式（+5/+10 按钮） |
| 职业选择 | 搜索框 + 列表 |
| 背景故事 | 引导式字段（6 个核心要素） |
| 装备物品 | 职业预设 + 自定义添加 |
| 页面操作 | 保存草稿 + 创建角色 |

## 页面结构

### 区块排列顺序

1. 基本信息
2. 核心属性 + 衍生属性
3. 技能分配
4. 背景故事
5. 装备物品

---

## 第一部分：整体布局与风格

### 页面结构

- 顶部固定 Header（与 Dashboard 一致：Monika 标题 + 用户信息）
- 主体内容区域 `container mx-auto px-4 py-8`
- 各区块使用 `Card` 组件包裹，区块间用 `space-y-6` 分隔

### 配色与风格

- 背景：`bg-background`（白色/透明，无渐变）
- 文字：主文字黑色，次级文字 `text-muted-foreground`
- 边框：`border` 简单边框
- 按钮：`Button` 组件，`variant="outline"` 或 `variant="default"`
- 无装饰性元素（无咖啡渍、无纸张纹理）

### 页面标题

```
创建调查员
Call of Cthulhu 7th Edition
```

## 第二部分：基本信息与核心属性

### 基本信息区块（Card 组件）

```
┌─────────────────────────────────────────────────────┐
│ 基本信息                                              │
├─────────────────────────────────────────────────────┤
│ 姓名    [________________]    年龄    [____]         │
│ 性别    ○ 男  ○ 女  ○ 其他     职业    [选择职业 ↓]   │
└─────────────────────────────────────────────────────┘
```

- 姓名、年龄：简单输入框
- 性别：单选按钮组
- 职业：点击后弹出职业选择面板（搜索框 + 列表）

### 核心属性区块（Card 组件）

```
┌───────────────────────────────────────────────────────────────┐
│ 核心属性                                        [一键生成数值]  │
├───────────────────────────────────────────────────────────────┤
│  力量 STR  [ __ ] 🎲   体质 CON  [ __ ] 🎲   体型 SIZ  [ __ ] 🎲 │
│  敏捷 DEX  [ __ ] 🎲   外貌 APP  [ __ ] 🎲   智力 INT  [ __ ] 🎲 │
│  意志 POW  [ __ ] 🎲   教育 EDU  [ __ ] 🎲                         │
├───────────────────────────────────────────────────────────────┤
│ 衍生属性                                                        │
│  HP [ __ ]  MP [ __ ]  SAN [ __ ]/99  移动 [ __ ]             │
│  体格 [ __ ]  伤害加成 [ ____ ]  幸运 [ __ ] 🎲                  │
└───────────────────────────────────────────────────────────────┘
```

- 每个属性右侧有 🎲 按钮，点击直接生成该属性值（无动画）
- 幸运值也可单独投骰
- "一键生成数值" 批量生成所有属性
- 衍生属性自动计算：
  - HP = (CON + SIZ) ÷ 10
  - MP = POW ÷ 5
  - SAN = POW（最大 99）
  - 移动力：根据 STR/DEX/SIZ 计算
  - 体格/伤害加成：根据 STR+SIZ 计算

│ ┌───────────────────────────────────────────────────────────┐ │
│ │ 心理学      基础 10   [ - ] [ 50 ] [ + ]  [+10]           │ │
│ │ 洞察        基础 25   [ - ] [ 45 ] [ + ]  [+10]           │ │
│ │ 图书馆使用  基础 25   [ - ] [ 65 ] [ + ]  [+10]           │ │
│ │ ...                                                       │ │
│ └───────────────────────────────────────────────────────────┘ │
├───────────────────────────────────────────────────────────────┤
│ 兴趣技能（剩余点数：___）                                       │
│ ┌───────────────────────────────────────────────────────────┐ │
│ │ [🔍 搜索技能...                    ] [添加技能]           │ │

- 每个技能：`[-]` 减5，`[+]` 加5，`[+10]` 加10
- 兴趣技能：需搜索添加，不自动显示

## 第五部分：页面操作与数据流

### 底部操作区

```
┌───────────────────────────────────────────────────────────────┐
│                                                   [保存草稿] [创建角色] │
└───────────────────────────────────────────────────────────────┘
```

- **保存草稿**：`variant="outline"`，保存当前数据到草稿，可稍后继续编辑
- **创建角色**：`variant="default"`，验证必填字段后正式创建角色

### 必填字段验证

| 字段 | 验证规则 |
|------|----------|
| 姓名 | 必填 |
| 年龄 | 必填，15-90 范围 |
| 职业 | 必选 |
| 所有属性 | 必须已投骰生成（值 > 0） |
| 背景故事 | 六个字段至少各填写 10 个字符 |

### 数据流

```
前端状态 (useState)
    ↓
保存草稿 → localStorage + API (status: draft)
    ↓
创建角色 → 验证 → API (status: active) → 跳转角色列表
```

### API 调用

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/occupations` | 获取职业列表 |
| POST | `/characters` | 创建角色 |
| PUT | `/characters/{id}` | 更新草稿 |

## 实现注意事项

1. **状态管理**：使用 useState 管理整个表单状态，考虑使用 useReducer 如果状态逻辑复杂
2. **表单验证**：创建角色前验证所有必填字段
3. **衍生属性计算**：属性变化时自动重新计算 HP/MP/SAN/移动力/体格/伤害加成
4. **职业联动**：选择职业后自动填充职业技能列表和基础装备
5. **草稿保存**：保存到 localStorage 作为备份，同时调用 API 保存到服务器
6. **响应式布局**：属性卡片在移动端显示为 2 列，桌面端显示为 4 列
