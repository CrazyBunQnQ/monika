# 命令集规范

**版本**: v1.0
**最后更新**: 2026-02-07
**状态**: ✅ 设计完成

---

## 概述

本文档定义 CoC 跑团平台的命令系统规范，包括命令语法、别名映射、输出格式和错误处理机制。

**设计原则**:
- **渐进式**: 基础简洁，高级灵活
- **自然语言**: 接近日常表达
- **叙事优先**: 沉浸式游戏体验
- **智能引导**: 友好的错误处理

---

## 核心命令清单

### 基础命令 (5个)

| 命令 | 功能 | 示例 | 类别 |
|------|------|------|------|
| `/help [命令]` | 显示帮助信息 | `/help`, `/help roll` | 基础 |
| `/status` | 显示当前状态 | `/status` | 基础 |
| `/leads` | 显示可选行动 | `/leads` | 基础 |
| `/rule <查询>` | 规则问答 | `/rule 暗视` | 基础 |
| `/quit` | 结束当前会话 | `/quit` | 基础 |

### 检定命令 (4个)

| 命令 | 功能 | 示例 | 类别 |
|------|------|------|------|
| `/roll <技能>` | 技能/属性检定 | `/roll 图书馆使用`, `/roll DEX` | 检定 |
| `/push` | 推骰（失败后可重掷） | `/push` | 检定 |
| `/luck [n]` | 花幸运（需引用事件） | `/luck`, `/luck 3` | 检定 |
| `/diff <等级>` | 设置检定难度（KP专用） | `/diff 困难` | 检定 |

### 战斗命令 (3个)

| 命令 | 功能 | 示例 | 类别 |
|------|------|------|------|
| `/combat start` | 开始战斗 | `/combat start` | 战斗 |
| `/combat action <动作>` | 执行战斗动作 | `/combat attack`, `/combat dodge` | 战斗 |
| `/combat end` | 结束战斗 | `/combat end` | 战斗 |

### 状态命令 (3个)

| 命令 | 功能 | 示例 | 类别 |
|------|------|------|------|
| `/san check <值>` | SAN 检定 | `/san check 10` | 状态 |
| `/heal [n]` | 治疗 | `/heal`, `/heal 3` | 状态 |
| `/rest <类型>` | 休息恢复 | `/rest 短休`, `/rest 长休` | 状态 |

## 命令语法设计

### 基础语法（新手友好）

```
/roll <技能名>
/combat start
/status
```

### 扩展语法（高级用户）

```
/roll <技能名> + <奖励骰>
/roll <技能名> - <惩罚骰>
/combat start --initiative
```

### 自然语言变体

```
/roll library use          # 完整自然
/roll library_use          # 下划线连接
/roll 图书馆使用            # 中文
/r 图书馆                  # 别名+简写
检定 图书馆                # 中文别名
```

### 参数顺序灵活

```
/roll library_use with bonus   # 自然
/roll library_use +2           # 简洁
/roll +2 library_use           # 也支持
```

## 命令输出格式

### 标准输出结构

每次命令执行后，输出分为三部分：

#### 1. 叙事段落（AI 生成）

```
你仔细翻阅着这本泛黄的古籍，尘埃在光线中飞舞。随着你逐页查
找，关于那个神秘教派的线索逐渐浮现……
```

#### 2. 结构化结果

```
【检定结果】图书馆使用
├─ 掷骰: 78 / 100
├─ 成功值: 60%
├─ 难度修正: 无
└─ 最终判定: 成功 ✅ (普通成功)

状态变更:
├─ 获得线索: [神秘教派的记载]
└─ Leads 刷新: +2 个新选项
```

#### 3. 下一步提示（可选）

```
💡 你现在可以：
• /leads 查看可选行动
• /roll 调查 继续调查
• /rule 图书馆使用 查看技能说明
```

### 特殊输出

#### 失败但可推骰

```
...你未能找到有用的信息。

【检定结果】图书馆使用 - 失败 ❌
你可以尝试 /push 推骰重来一次（消耗幸运）
```

#### 大成功/大失败

```
...不仅找到了记载，你还发现了一张夹在书页间的旧照片！🌟

【检定结果】图书馆使用 - 大成功 🌟
（掷骰 ≤ 5）
额外奖励: 获得物品 [旧照片]
```

## 错误处理

### 无法识别的命令

```
❓ 不太确定你的意思...

你是想：
→ /combat goblin  开始战斗
  (fight 是 combat 的别名)

其他相关命令：
• /roll spot_hidden  侦查检定
• /help combat       查看战斗帮助

输入 /help 查看所有命令
```

### 缺少必要参数

```
> /roll

❓ 你想检定什么？

可用技能：
• 图书馆使用 (60%)  • 侦查 (50%)  • 聆听 (40%)

或者：
• /roll STR  属性检定
• /roll --help  查看帮助
```

### 参数值无效

```
> /roll 飞行

❌ "飞行" 不是有效的技能

最接近的技能：
• 跳跃 (25%)  • 攀爬 (30%)

输入 /skills 查看所有技能
```

### 上下文错误

```
> /combat attack

⚠️ 当前不在战斗中

可用操作：
• /combat start  开始战斗
• /roll <技能>   进行检定
```

## 系统架构

### 三层架构

```
┌─────────────────────────────────────────────┐
│              用户输入                         │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│           解析层 (Parser)                    │
│  • 自然语言解析                               │
│  • 别名映射                                   │
│  • 参数提取                                   │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│           执行层 (Executor)                  │
│  • 命令处理器                                 │
│  • 游戏状态交互                               │
│  • 返回执行结果                               │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│           呈现层 (Presenter)                 │
│  • 叙事生成 (AI)                             │
│  • 格式化输出                                 │
│  • 可见性控制                                 │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│              用户输出                         │
└─────────────────────────────────────────────┘
```

### 解析流程

```
用户输入: "r 图书馆 +2"
    ↓
1. 分词: ['r', '图书馆', '+2']
    ↓
2. 命令映射: 'r' → '/roll'
    ↓
3. 技能映射: '图书馆' → 'library_use'
    ↓
4. 参数解析: '+2' → {modifier: '+2'}
    ↓
标准化: /roll library_use +2
```
