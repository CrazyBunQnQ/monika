# Configuration Reference

Complete reference for all environment variables in Monika.

## Quick Setup

**Minimum required configuration** (3 LLM settings + 1 security):

```bash
# .env file
SECRET_KEY=<auto-generated>

# LLM Configuration (choose one provider)
LLM_PROVIDER=openai
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_API_KEY=sk-your-key-here
OPENAI_MODEL=gpt-4o-mini
```

---

## Required Variables

### Security

| Variable | Description | Example |
|----------|-------------|---------|
| `SECRET_KEY` | JWT signing key (min 32 chars) | `abc123...` (use `openssl rand -hex 32`) |

### LLM Configuration (Required)

Choose **one** provider and configure all three settings:

#### OpenAI

| Variable | Description | Example |
|----------|-------------|---------|
| `OPENAI_BASE_URL` | API endpoint URL | `https://api.openai.com/v1` |
| `OPENAI_API_KEY` | Your API key | `sk-...` |
| `OPENAI_MODEL` | Model name | `gpt-4o-mini` |

#### Claude

| Variable | Description | Example |
|----------|-------------|---------|
| `CLAUDE_BASE_URL` | API endpoint URL | `https://api.anthropic.com` |
| `CLAUDE_API_KEY` | Your API key | `sk-ant-...` |
| `CLAUDE_MODEL` | Model name | `claude-3-5-sonnet-20241022` |

## Database Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `DB_HOST` | `postgres` | Database host (Docker service name) |
| `DB_PORT` | `5432` | Database port |
| `DB_NAME` | `monika` | Database name |
| `DB_USER` | `postgres` | Database user |
| `DB_PASSWORD` | `postgres` | Database password |

**Note**: For Docker Compose, use the defaults. For external database, update these values.

---

## Optional Settings

| Variable | Default | Description |
|----------|---------|-------------|
| `DEBUG` | `false` | Enable debug mode (always false in production) |
| `LLM_PROVIDER` | `openai` | Which LLM to use: `openai` or `claude` |

```bash

OPENAI_BASE_URL=https://api.openai.com/v1
```

## Security Best Practices

1. **Never commit .env to version control** - It's already in .gitignore
2. **Use strong random keys** - `openssl rand -hex 32`
3. **Use different keys for each environment** - dev/staging/prod
4. **Rotate keys periodically** - Every 90 days recommended
5. **Use secrets management in production** - Docker Secrets, AWS Secrets Manager, etc.

## Generating Secure Keys

```bash
# Generate SECRET_KEY
openssl rand -hex 32

# Generate DB_PASSWORD
openssl rand -base64 16 | tr -d "=+/" | cut -c1-16
```

The deployment script (`deploy.sh`) does this automatically!

## Troubleshooting

### LLM Connection Failed

**Error**: `Failed to connect to LLM API`

**Solutions**:
1. Verify `BASE_URL` is correct
2. Check `API_KEY` is valid
3. Ensure `MODEL` name is correct
4. Test API key with provider's dashboard

### Invalid SECRET_KEY

**Error**: `SECRET_KEY must be at least 32 characters`

**Solution**: Generate with `openssl rand -hex 32`

### Database Connection Failed

**Error**: `Could not connect to database`

**Solution**:
- Docker Compose: Check `DB_HOST=postgres`
- External DB: Verify `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`
