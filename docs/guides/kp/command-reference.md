## 基础命令

### /help - 显示帮助

**语法**: `/help [command]`

**说明**: 显示帮助信息。带参数时显示指定命令的详细说明。

**参数**:
- `command` (可选): 要查询的命令名称

**示例**:
```bash
/help           # 显示所有命令
/help roll      # 显示 /roll 命令的详细说明
```

**权限**: 所有角色

---

### /status - 显示状态

**语法**: `/status [character_id]`

**说明**: 显示角色状态。不带参数显示所有角色，带参数显示指定角色。

**输出内容**:
- HP / HP_max
- SAN / SAN_max
- Luck / Luck_max
- 当前状态条件
- 技能列表（摘要）

**示例**:
```bash
/status              # 显示所有角色状态
/status player_001   # 显示玩家 001 的状态
```

**权限**: KP / 玩家（自己的角色）

---

### /leads - 显示可选行动

**语法**: `/leads [refresh]`

**说明**: 显示当前可选行动列表。

**参数**:
- `refresh` (可选): 强制刷新列表

**输出**:
- 可用行动标题
- 行动描述
- 执行方式
- 优先级标识

**示例**:
```bash
/leads           # 显示可选行动
/leads refresh   # 强制刷新
```

**权限**: 所有角色

---

### /rule - 规则问答

**语法**: `/rule <查询>`

**说明**: 查询 CoC 7th Edition 规则。

**示例**:
```bash
/rule 暗视              # 查询暗视技能
/rule 如何计算伤害      # 查询伤害计算规则
/rule 闪避              # 查询闪避规则
```

**权限**: 所有角色

---

### /quit - 结束会话

**语法**: `/quit`

**说明**: 结束当前跑团会话。

**确认**: 执行前会要求 KP 确认

**示例**:
```bash
/quit
# 系统提示: 确定要结束会话吗？(yes/no)
```

**权限**: 仅 KP

## 检定命令

### /roll - 技能检定

**语法**: `/roll <技能名> [修饰符]`

**说明**: 执行技能或属性检定。

**参数**:
- `技能名`: 检定的技能名称，支持中英文
- `修饰符` (可选):
  - `+n`: 奖励骰
  - `-n`: 惩罚骰
  - `difficulty <级别>`: 设置难度

**难度级别**:
- `easy`: 简单
- `regular`: 普通（默认）
- `hard`: 困难
- `extreme`: 极难
- `critical`: 大成功要求

**示例**:
```bash
/roll library_use               # 图书馆使用检定
/roll 图书馆使用                 # 中文技能名
/roll spot_hidden +1             # 带奖励骰
/roll DEX -1                     # 属性检定带惩罚骰
/roll drive difficulty hard       # 设置困难难度
```

**成功等级**:
| 掷骰 | 成功值 | 结果 |
|------|--------|------|
| ≤ 5 | 任意 | 大成功 🌟 |
| ≤ 成功值/2 | 困难 | 极难成功 ✨ |
| ≤ 成功值/5 | 极难 | 困难成功 👍 |
| ≤ 成功值 | 普通 | 普通成功 ✅ |
| > 成功值 | 普通 | 失败 ❌ |
| 100 | 任意 | 大失败 💀 |

**权限**: 所有角色

---

### /push - 推骰

**语法**: `/push`

**说明**: 重新执行上一次失败的检定（推骰）。

**限制**:
- 只能在检定失败后使用
- 需要消耗幸运值
- 每次检定只能推一次

**示例**:
```bash
/roll spot_hidden
# 结果: 失败 (85/50)
/push
# 重新掷骰，消耗幸运 5 点
# 新结果: 成功 (32/50) ✅
```

**权限**: 玩家（自己的角色）

---

### /luck - 花幸运

**语法**: `/luck [n] [事件引用]`

**说明**: 消耗幸运值改变某次检定结果。

**参数**:
- `n` (可选): 花费的幸运点数，默认自动计算
- `事件引用` (可选): 要改变的事件，默认是上一次检定

**规则**:
- 每点幸运可将掷骰结果 ±5
- 可以花费到幸运值降为 0
- 花幸运后的结果不能再次修改

**示例**:
```bash
/roll spot_hidden
# 结果: 大失败 (100/50) 💀
/luck 10 "spot_hidden"
# 花费 10 点幸运，掷骰结果 -50
# 新结果: 50 = 成功值，普通成功 ✅
```

**权限**: 玩家（自己的角色）

## 战斗命令

### /combat start - 开始战斗

**语法**: `/combat start`

**说明**: 开始战斗模式，初始化先攻轮。

**效果**:
- 所有参与者投先攻（DEX 检定）
- 按先攻值排序确定行动顺序
- 进入战斗回合循环

**示例**:
```bash
/combat start
# 战斗开始！
# 先攻顺序: 玩家A (12) → 玩家B (9) → 哥布林 (8)
```

**权限**: KP

---

### /combat action - 战斗动作

**语法**: `/combat action <动作> [目标]`

**说明**: 执行战斗动作。

**可用动作**:
- `attack <目标>`: 攻击
- `dodge`: 闪避
- `maneuver <战技>`: 战技（高级规则）
- `flee`: 逃跑

**示例**:
```bash
/combat action attack npc_goblin    # 攻击哥布林
/combat action dodge               # 闪避
/combat action flee                # 逃跑
```

**权限**: 所有战斗参与者

---

### /combat end - 结束战斗

**语法**: `/combat end`

**说明**: 结束当前战斗。

**效果**:
- 退出战斗模式
- 结算战斗奖励
- 记录战斗事件

**示例**:
```bash
/combat end
# 战斗结束！
# 战利品: 获得金币 x10
# 经验: +50 XP
```

**权限**: KP

## 状态命令

### /san - SAN 相关

**语法**: `/san <子命令> [参数]`

**子命令**:

#### /san check <值> - SAN 检定
**说明**: 触发 SAN 检定。

**参数**:
- `值`: SAN 损失值

**示例**:
```bash
/san check 10      # 进行 10 点 SAN 检定
/san check 5       # 进行 5 点 SAN 检定
```

#### /san heal <n> - 恢复 SAN
**说明**: 恢复角色的 SAN 值。

**参数**:
- `n`: 恢复的点数

**限制**:
- SAN 不能超过当前最大值
- 需要合理的剧情理由

**示例**:
```bash
/san heal 5        # 恢复 5 点 SAN
```

**权限**: KP

---

### /heal - 治疗

**语法**: `/heal [n] [角色]`

**说明**: 治疗角色的 HP。

**参数**:
- `n` (可选): 治疗的点数，默认为 1d6
- `角色` (可选): 目标角色，默认为当前角色

**示例**:
```bash
/heal                    # 治疗 1d6 点
/heal 5                  # 治疗 5 点
/heal player_001         # 治疗指定角色
```

**权限**: KP / 玩家（自己的角色，需合理理由）

---

### /rest - 休息

**语法**: `/rest <类型>`

**说明**: 角色休息恢复。

**休息类型**:
- `short`: 短休（1小时）
- `long`: 长休（8小时）
- `full`: 完全休息（24小时）

**效果**:
- 恢复部分 HP
- 恢复部分 MP
- 结束某些状态条件

**示例**:
```bash
/rest short         # 短休
/rest long          # 长休
```

**权限**: KP

## KP 专用命令

### /diff - 设置检定难度

**语法**: `/diff <难度>`

**说明**: 设置下一次检定的难度等级。

**难度级别**:
- `easy`: 简单（检定值 ×2）
- `regular`: 普通（默认）
- `hard`: 困难（检定值 ÷2）
- `extreme`: 极难（检定值 ÷5）

**示例**:
```bash
/diff hard        # 设置为困难难度
/roll spot_hidden # 之后的检定按困难难度计算
```

**权限**: 仅 KP

## 常见问题

### Q: 如何查看所有可用技能？

A: 使用 `/rule 技能列表` 查看完整技能列表。

### Q: 推骰需要消耗什么？

A: 推骰需要消耗幸运值，具体数量由规则决定。

### Q: 如何设置自定义难度？

A: 使用 `/diff <难度>` 命令，然后执行检定。
