# CoC 跑团平台 - 文档中心

## 文档结构

本项目的产品和技术文档组织如下：

```
docs/
├── prd/
│   ├── PRD.md                 # 产品需求文档（主文档）
│   ├── detailed-specs.md      # 详细技术规格（M0-M6）
```

## 文档说明

### PRD.md - 产品需求文档

**用途**：产品的总体定义和规划文档

**包含内容**：
- 产品概述与定位
- 目标用户与使用场景
- 核心价值主张
- 产品范围（功能边界）
- 功能需求（按用户价值组织）
- 非功能需求（性能/安全/可维护性）
- 用户体验设计原则
- 数据模型概览
- 里程碑规划
- 成功指标
- Web 应用架构
- 技术栈说明

**适用人群**：
- 产品经理
- 项目经理
- 开发团队
- 设计团队

### detailed-specs.md - 详细技术规格

**用途**：开发实施的技术依据，包含所有里程碑的详细规格

**包含内容**：
- **M0：冻结规范**
  - 最小用户命令清单
  - 模组场景包格式
  - 状态字段清单
- **M1：单人单场景可玩循环**
  - 单人场景循环
  - 检定闭环
  - 战斗闭环
  - 追逐闭环
  - 规则问答闭环
- **M2：多人桌面流程**
  - 多人加入/身份/席位
  - 聚光灯/队列
  - 可见性与信息泄露防护
  - 多人状态查询与复盘
  - 共享状态写入与所有权
  - 多人战斗/追逐联动
  - 暂停/安全工具
  - 多人鲁棒性
- **M3：长团记忆**
  - 事件日志与索引
  - 结构化摘要
  - 记忆检索与引用
  - 恢复与一致性
- **M4：模组/规则书知识库**
  - 模组导入与校验
  - 规则书入库
  - 统一检索接口
  - 剧情推进的模组落地
  - 访问控制与防注入
- **M5：CoC 全功能补齐**
  - SAN/疯狂机制
  - 战斗完整流程
  - 追逐补齐
  - 成长与战后流程
  - 端到端短模组验收
- **M6：体验打磨**
  - Leads 面板与线索账本
  - 防卡死与失败前进
  - 越界拒绝与命令引导
  - 输出稳定性与信息密度
- **附录**：数据结构总览

**适用人群**：
- 开发人员
- 测试人员
- 技术架构师

## 文档使用指南

### 对于产品经理

1. 从 **PRD.md** 开始，了解产品全貌
2. 查看"里程碑规划"了解开发节奏
3. 参考"功能需求"了解各功能模块的用户价值

### 对于开发人员

1. 先阅读 **PRD.md** 了解产品定位和技术栈
2. 然后深入 **detailed-specs.md** 查看具体技术规格
3. 按里程碑顺序（M0 → M1 → M2 → ...）实施开发
4. 参考详细规格中的数据结构和验收标准

### 对于测试人员

1. 参考 **PRD.md** 中的"成功指标"制定测试策略
2. 使用 **detailed-specs.md** 中的验收标准编写测试用例
3. 每个里程碑完成后，对照详细规格中的验收标准进行验证

### 对于设计师

1. 参考 **PRD.md** 中的"用户体验设计原则"
2. 查看"Web 应用架构"了解页面结构和组件需求
3. 参考"输出格式标准"了解界面展示要求

## 文档维护

### 更新原则

- **PRD.md**：产品层面的变更（功能增减、里程碑调整、成功指标变更等）
- **detailed-specs.md**：技术实现细节的变更（数据结构、接口定义、验收标准等）

### 版本控制

两个文档都有独立的版本号和变更记录（见各自文档的附录 D）。

### 变更流程

1. 确定变更属于哪个文档
2. 更新相应文档的内容
3. 在"变更记录"中记录变更内容
4. 提交 PR 评审

## 技术栈总览

详细的技术栈说明请参考 [PRD - 第 4.3 节](../PRD.md#43-技术架构)。

### 快速参考

- **前端**: React 18 + shadcn/ui + Zustand + TypeScript
- **后端**: Python + Agno 框架 + FastAPI
- **数据库**: PostgreSQL + Redis
- **AI**: Agno 内置 LLM 集成（支持 OpenAI/Claude）
- **部署**: Docker + 云服务
