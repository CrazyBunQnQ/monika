## 任务标记说明

| 标记 | 含义 | 说明 |
|------|------|------|
| [ ] | 待开始 | 任务未开始 |
| [P] | 进行中 | 任务正在执行 |
| [D] | 待审核 | 任务完成，等待 Code Review |
| [✅] | 已完成 | 任务验收通过 |

---

## 任务 ID 命名规则

```
[M][里程碑]-[序号]

示例:
M0-001  -> M0 里程碑第 1 个任务
M1-012  -> M1 里程碑第 12 个任务
M2-076  -> M2 里程碑第 76 个任务
```

## 任务类型说明

| 类型 | 含义 | 说明 |
|------|------|------|
| spec | 规范设计 | 定义接口、数据结构 |
| db | 数据库设计 | 表结构、迁移脚本 |
| backend | 后端开发 | API、业务逻辑 |
| frontend | 前端开发 | UI、组件、交互 |
| infra | 基础设施 | 环境、部署、配置 |
| test | 测试 | 单元测试、集成测试 |
| design | 设计 | UI/UX 设计规范 |
| doc | 文档 | 文档编写 |
| plan | 规划 | 技术方案设计 |
