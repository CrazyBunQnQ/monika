Dockview 验证结果

兼容 React 18.x | v5.2.0 | MIT | 零外部依赖

关键发现

# 验证项 结果 说明
1 自定义 Tab 标题栏
放 ModelPicker、状态指示器、关闭按钮		 ✅ 可行 IDockviewPanelHeaderProps 提供 api、containerApi、params、tabLocation。
每个 Panel 可指定 tabComponent，或全局 defaultTabComponent。
可继承 DockviewDefaultTab 添加自定义元素。
⚠️ api.setTitle() 对自定义 Tab 有 bug (#1003)，用 React state 绕过即可。
2 Zustand 状态管理集成
不替换现有 store，嵌入 dockview		 ✅ 原生兼容 Panel 内容组件就是普通 React 组件，直接用 useStore() hook。
Zustand 作为 external store 完全独立于 dockview 的序列化生命周期。
params 仅用于静态序列化数据，动态状态走 Zustand。
社区已有实践案例（AutoArt SPA 等）。
3 Wails WebView2 兼容性
Chromium 内核 + CSS 自定义属性		 ✅ 无问题 WebView2 基于 Chromium，完全支持 CSS var()、Flexbox、Grid。
dockview 主题系统用 --dv-* CSS 变量，无需 polyfill。
dockview-core 零依赖纯 TypeScript。
⚠️ 建议测试拖拽 adorner 在 WebView2 中的渲染表现。
4 CodeMirror 实例持久化
Tab 切换不销毁编辑器		 ✅ 有方案 Panel 可用 renderer: 'always' 保持 DOM 不销毁。
Tab 切换时只是 display:none，CodeMirror EditorView 实例存活。
编辑器状态存 Zustand store，内容持久化不受布局序列化影响。
onDidVisibilityChange 事件可按需优化。

API 架构适配分析

Monika 现有组件
App.tsx → 硬编码布局
├─ SessionList (Sidebar)
├─ ChatArea (TabBar + Messages)
├─ FileEditor (TabBar + CodeMirror)
├─ FileTree (Sidebar)
└─ Console (Header + Log)
Zustand Store
├─ openSessions / messages
├─ openFiles / fileContent
└─ layoutMode / splitRatio
Dockview 映射
<DockviewReact>
components 映射：
'chat' → ChatArea
'editor' → FileEditor
'filetree' → FileTree
'session' → SessionList
'console' → Console
tabComponents 映射：
自定义 Tab（ModelPicker 等）
各 Panel 内通过 Zustand 读/写状态
api.toJSON() → 布局持久化

注意事项

优势

  • 零依赖，不污染 node_modules
  • React 18.x peerDependency 完全匹配
  • Tab 拖拽重组、浮动面板、Popout 开箱即用
  • 布局 JSON 序列化/反序列化（可持久化到 session）
  • dockview-core 引擎 + React wrapper 分离，架构清晰
  • 8 套内置主题，CSS 变量可覆盖
  • ~61KB minified，不算重

需要处理的问题

  • api.setTitle() 对自定义 Tab 不触发更新 → 从 Zustand 读 title
  • params 更新不触发 Tab 重渲染 → 动态数据走 Zustand
  • WebView2 拖拽 adorner 需要实测确认
  • Console 的垂直 resize 可能需要独立处理（dockview 主要用于水平+垂直分割）
  • 关闭 Tab 前的 dirty 检查（ConfirmModal）需要 Hook dockview 的关闭事件
