设计要点

后端 ListDir() 先跑一次 git status --porcelain 构建 path→status 映射，递归填充 FileNode.Status
文件树 前端已有 gitColor() 和状态标签逻辑，后端填数据即工作
模式切换 TabBar 右侧新增 Edit | Diff 分段按钮
编辑模式 CodeMirror 可编辑，Ctrl+S 保存，复用已有 isDirty 状态
Diff 模式 调用 GetFileDiff()，统一 diff 格式，+绿色背景 / -红色背景
Diff 范围 工作区 vs HEAD（git diff <file>）
